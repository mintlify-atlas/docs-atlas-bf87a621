The DIM project includes a complete sample style guide that demonstrates how to create an intelligent, enforceable style guide using DITA and embedded Schematron rules.

## Overview

The sample style guide was contributed by [Comtech Services](http://www.comtech-serv.com) and provides comprehensive authoring guidelines for DITA content. It serves as both a functional style guide and a template for creating your own DIM-enabled documentation standards.

<Note>
This style guide can be used as a starting point for your own organization's style guide, or you can integrate DIM features into your existing DITA-based documentation.
</Note>

## Sample Topics

### Titling Topics (c_TitlingTopics.dita)

A comprehensive guide to creating effective titles in DITA.

<Accordion title="Topic Structure">

**Content Guidelines**:
- Use specific, descriptive language
- Limit titles to 8 words or less
- Use title case capitalization
- Avoid generic terms like "Introduction"
- Include context-specific information

**Information Type Guidelines**:
- **Task titles**: Use imperative verb forms
- **Concept titles**: Use noun phrases
- **Reference titles**: Include the referenced item name

**Embedded Rules**:

1. **Restrict Children**: Only `ph` elements allowed in titles
   ```xml
   <dl>
     <dlhead>
       <dthd>Rule</dthd>
       <ddhd>restrictChildrenElements</ddhd>
     </dlhead>
     <dlentry>
       <dt>parentElement</dt>
       <dd>title</dd>
     </dlentry>
     <dlentry>
       <dt>allowedChildren</dt>
       <dd>ph</dd>
     </dlentry>
     <dlentry>
       <dt>message</dt>
       <dd>Only "ph" is allowed within a title element.</dd>
     </dlentry>
   </dl>
   ```

2. **Word Count**: Titles should be 1-8 words
   ```xml
   <dl>
     <dlhead>
       <dthd>Rule</dthd>
       <ddhd>restrictWords</ddhd>
     </dlhead>
     <dlentry>
       <dt>parentElement</dt>
       <dd>title</dd>
     </dlentry>
     <dlentry>
       <dt>minWords</dt>
       <dd>1</dd>
     </dlentry>
     <dlentry>
       <dt>maxWords</dt>
       <dd>8</dd>
     </dlentry>
     <dlentry>
       <dt>message</dt>
       <dd>Keep titles between 1 and 8 words.</dd>
     </dlentry>
   </dl>
   ```

</Accordion>

## Generated Files

Running the XSLT scripts on the sample style guide generates:

### rules.sch

A complete Schematron schema with all embedded rules instantiated:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!--
  Do not edit this file directly!
  This file is generated automatically by processing styleguide.ditamap
-->
<schema xmlns="http://purl.oclc.org/dsdl/schematron" queryBinding="xslt2">
  
  <!-- Library pattern includes -->
  <include href="library.sch#avoidWordInElement"/>
  <include href="library.sch#restrictWords"/>
  <!-- ... more includes ... -->
  
  <include href="quickFix-library.xml"/>
  
  <!-- Pattern instantiations from topics -->
  <!--Generated from info-model/c_TitlingTopics.dita.-->
  <pattern is-a="restrictChildrenElements" 
           see="http://example.com/styleguide/webhelp/info-model/c_TitlingTopics.html">
    <param name="parentElement" value="title"/>
    <param name="allowedChildren" value="ph"/>
    <param name="message" value="Only 'ph' is allowed within a title element."/>
  </pattern>
  
  <!-- ... more patterns ... -->
</schema>
```

### contentCompletionElementsMap.xml

Element and attribute documentation mappings:

```xml
<contentCompletionElementsMap
  redirectURLPrefix="http://www.oxygenxml.com/redirect-url.php?url="
  htmlContentFilterStylesheet="contentFilter.xsl">
  
  <mappingGroup xml:base="http://example.com/styleguide/webhelp/">
    <mapping elementName="title" 
             path="info-model/c_TitlingTopics.html" 
             type="link" 
             linkText="Titling Topics"/>
    <mapping elementName="shortdesc" 
             path="info-model/c_WritingShortDescriptions.html" 
             type="link" 
             linkText="Writing Short Descriptions"/>
    <!-- ... more mappings ... -->
  </mappingGroup>
</contentCompletionElementsMap>
```

## Customization Guide

### Using as a Template

<Steps>
  <Step title="Clone or Copy">
    Start with the sample style guide as a foundation:
    ```bash
    cp -r info-model my-styleguide
    ```
  </Step>
  
  <Step title="Update Content">
    - Modify existing topics to match your standards
    - Add new topics for organization-specific guidelines
    - Update the bookmap structure
  </Step>
  
  <Step title="Adjust Rules">
    - Change parameter values to match your requirements
    - Add new rule instantiations
    - Remove rules that don't apply
  </Step>
  
  <Step title="Update Metadata">
    - Add metadata annotations for your elements
    - Link attributes to your documentation
    - Update company name and branding
  </Step>
  
  <Step title="Generate and Test">
    - Run gen-rules.xsl and gen-mappings.xsl
    - Publish to WebHelp
    - Test validation with sample content
  </Step>
</Steps>

### Integrating with Existing Style Guides

If you have an existing DITA-based style guide:

<Steps>
  <Step title="Add Rules Sections">
    Insert `section[@audience='rules']` elements where you want to enforce guidelines.
  </Step>
  
  <Step title="Add Metadata">
    Include `prolog/metadata` with `data[@audience='styleguide']` annotations.
  </Step>
  
  <Step title="Reference Library">
    Copy `library.sch` to your project and reference it in the XSLT scripts.
  </Step>
  
  <Step title="Generate">
    Run the generation scripts on your existing map.
  </Step>
</Steps>

## Key Features Demonstrated

### 1. Dual-Purpose Content

Topics serve as both:
- **Human-readable documentation** (published to WebHelp)
- **Machine-processable rules** (extracted to Schematron)

### 2. Traceability

Every validation message links back to its source:
```xml
<pattern is-a="..." see="http://example.com/styleguide/.../topic.html">
```

Authors can click validation messages to view the full guideline context.

### 3. Context-Sensitive Help

Metadata annotations enable IDE integration:
- Hover over an element → see style guide link
- Press F1 → open relevant style guide topic
- Content completion → shows guidelines inline

### 4. Centralized Maintenance

Update guidelines in one place:
- Edit the DITA topic
- Regenerate rules.sch
- Rules automatically reflect the changes

### 5. Validation and Quick Fixes

Rules include Schematron Quick Fix references:
```xml
<assert test="..." sqf:fix="restrictWords_setNew">
  Keep titles between 1 and 8 words.
</assert>
```

Compatible editors offer one-click remediation.

## Publishing the Style Guide

### Generate WebHelp

Use the pre-configured transformation scenario:

<Steps>
  <Step title="Open Map">
    Open `styleguide.ditamap` in Oxygen XML Editor.
  </Step>
  
  <Step title="Select Scenario">
    Choose **Generate Style Guide** from the transformation scenarios.
  </Step>
  
  <Step title="Run">
    Execute the scenario to generate WebHelp output to `info-model/styleguide/`.
  </Step>
  
  <Step title="Deploy">
    Deploy the WebHelp output to a web server or share locally.
  </Step>
</Steps>

### Update URLs

After publishing, update the URL prefix in:

1. **gen-rules.xsl**:
   ```xml
   <xsl:variable name="target">
     <xsl:text>http://your-server.com/styleguide/webhelp/</xsl:text>
     <xsl:value-of select="replace($origin, '.dita', '.html')"/>
   </xsl:variable>
   ```

2. **gen-mappings.xsl**:
   ```xml
   <mappingGroup xml:base="http://your-server.com/styleguide/webhelp/">
   ```

3. **Catalog file** (`info-model/rules/catalog.xml`):
   ```xml
   <rewriteURI uriStartString="http://example.com/styleguide/webhelp/" 
               rewritePrefix="http://your-server.com/styleguide/webhelp/"/>
   ```

## Extending the Sample

Add new topics following the established patterns:

### Template for New Topics

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE concept PUBLIC "-//OASIS//DTD DITA Concept//EN" "concept.dtd">
<concept id="new_topic">
  <title>Your Topic Title</title>
  
  <prolog>
    <metadata>
      <!-- Add metadata links -->
      <data audience="styleguide" name="element-name" 
            value="Display text for link"/>
    </metadata>
  </prolog>
  
  <conbody>
    <p>Your content guidelines here...</p>
    
    <!-- Add rules section if needed -->
    <section audience="rules">
      <title>Business Rules</title>
      <p>Optional description of the rules.</p>
      
      <dl>
        <dlhead>
          <dthd>Rule</dthd>
          <ddhd>pattern-name</ddhd>
        </dlhead>
        <dlentry>
          <dt>parameter1</dt>
          <dd>value1</dd>
        </dlentry>
        <!-- More parameters -->
      </dl>
    </section>
  </conbody>
</concept>
```
