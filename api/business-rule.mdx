## BusinessRuleRepository

Singleton repository that loads and caches business rule libraries. Extracts rules from Schematron files using XSLT transformation.

### Methods

#### getInstance()

Returns the singleton instance of the repository.

<ResponseField name="return" type="BusinessRuleRepository">
  The singleton instance
</ResponseField>

```java
public static BusinessRuleRepository getInstance()
```

#### loadLibrary(AuthorXMLUtilAccess, URL)

Loads a rules library from a Schematron file, parsing and caching the results.

<ParamField path="authorXMLAccess" type="AuthorXMLUtilAccess" required>
  Oxygen XML utility access for XSLT transformation
</ParamField>

<ParamField path="library" type="URL" required>
  The URL to the Schematron library file
</ParamField>

<ResponseField name="return" type="BusinessRuleLibrary">
  The parsed library containing all extracted rules
</ResponseField>

<ResponseField name="throws" type="TransformerException">
  If XSLT transformation fails
</ResponseField>

<ResponseField name="throws" type="JAXBException">
  If XML parsing fails
</ResponseField>

```java
public BusinessRuleLibrary loadLibrary(
    AuthorXMLUtilAccess authorXMLAccess, 
    URL library
) throws TransformerException, JAXBException
```

### Example

```java
// Get the singleton instance
BusinessRuleRepository repository = BusinessRuleRepository.getInstance();

// Load a library
URL libraryURL = new URL("file:///path/to/rules.sch");
BusinessRuleLibrary library = repository.loadLibrary(
    authorAccess.getXMLUtilAccess(), 
    libraryURL
);

// Access cached rules
List<BusinessRule> rules = library.getRules();
```

### Caching Behavior

The repository maintains an internal cache of parsed libraries. When `loadLibrary()` is called:

1. It checks if the library is already cached
2. Compares the file's last modification timestamp
3. Reloads only if the file has changed
4. Returns the cached version if unchanged

This ensures efficient performance when the same library is accessed multiple times.
