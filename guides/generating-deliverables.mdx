## Overview

The DIM project generates three key deliverables from your DITA style guide:

1. **Schematron validation schema** (`rules.sch`) - Enforces your business rules
2. **Content completion mappings** (`contentCompletionElementsMap.xml`) - Links elements to style guide topics
3. **Published WebHelp** - Browser-accessible style guide documentation

These deliverables integrate into Oxygen XML Editor to provide an intelligent authoring experience.

## XSLT Transformation Scripts

DIM includes two primary XSLT scripts:

- **`gen-rules/gen-rules.xsl`** - Generates the Schematron schema from embedded business rules
- **`gen-rules/gen-mappings.xsl`** - Generates the Oxygen configuration file from metadata annotations

Both operate on your style guide DITA map as input.

## Generating the Schematron Schema

<Steps>
  <Step title="Ensure business rules are defined">
    Verify your style guide topics contain `<section audience="rules">` sections with rule definitions. See [Defining Rules](/guides/defining-rules) for details.
  </Step>

  <Step title="Run the gen-rules.xsl transformation">
    Execute the XSLT transformation using Saxon or another XSLT 2.0 processor:
    
    ```bash
    saxon -s:info-model/styleguide.ditamap \
          -xsl:gen-rules/gen-rules.xsl \
          -o:info-model/rules/rules.sch
    ```
    
    **Parameters:**
    - `-s` - Source: your style guide DITA map
    - `-xsl` - Stylesheet: the gen-rules.xsl script
    - `-o` - Output: the generated Schematron file (can be omitted; script writes to `rules/rules.sch` by default)
  </Step>

  <Step title="Review the generated schema">
    Open `info-model/rules/rules.sch` to see the compiled Schematron patterns:
    
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!-- Do not edit this file directly! -->
    <schema xmlns="http://purl.oclc.org/dsdl/schematron" queryBinding="xslt2">
      
      <include href="library.sch#restrictWords"/>
      <include href="library.sch#recommendElementInParent"/>
      
      <!-- Generated from info-model/c_TitlingTopics.dita -->
      <pattern is-a="restrictWords" 
               see="http://example.com/styleguide/webhelp/c_TitlingTopics.html">
        <param name="parentElement" value="title"/>
        <param name="minWords" value="1"/>
        <param name="maxWords" value="8"/>
        <param name="message" value="Keep titles between 1 and 8 words."/>
      </pattern>
      
    </schema>
    ```
    
    Each pattern includes a `see` attribute linking back to the source topic.
  </Step>
</Steps>

## Generating the Content Completion Map

<Steps>
  <Step title="Ensure topics are annotated">
    Verify your style guide topics contain `<data>` elements with `audience="styleguide"` in their metadata. See [Annotating Topics](/guides/annotating-topics) for details.
  </Step>

  <Step title="Run the gen-mappings.xsl transformation">
    Execute the transformation:
    
    ```bash
    saxon -s:info-model/styleguide.ditamap \
          -xsl:gen-rules/gen-mappings.xsl \
          -o:info-model/rules/contentCompletionElementsMap.xml
    ```
    
    **Parameters:**
    - `-s` - Source: your style guide DITA map
    - `-xsl` - Stylesheet: the gen-mappings.xsl script
    - `-o` - Output: the generated configuration file (can be omitted; script writes to `rules/contentCompletionElementsMap.xml` by default)
  </Step>

  <Step title="Review the generated configuration">
    Open `info-model/rules/contentCompletionElementsMap.xml`:
    
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!-- Do not edit this file directly! -->
    <contentCompletionElementsMap
        redirectURLPrefix="http://www.oxygenxml.com/redirect-url.php?url="
        htmlContentFilterStylesheet="contentFilter.xsl">
      <mappingGroup xml:base="http://example.com/styleguide/webhelp/">
        
        <mapping elementName="shortdesc" 
                 path="c_WritingShortDescriptions.html" 
                 type="link" 
                 linkText="How to write short descriptions"/>
        
        <mapping elementName="title" 
                 path="c_TitlingTopics.html" 
                 type="link" 
                 linkText="Guidelines for writing titles"/>
        
      </mappingGroup>
    </contentCompletionElementsMap>
    ```
  </Step>
</Steps>

## Publishing the WebHelp Output

The style guide must be published to WebHelp format for links from Oxygen to work.

<Steps>
  <Step title="Open the style guide map in Oxygen">
    Load `info-model/styleguide.ditamap` in Oxygen XML Editor.
  </Step>

  <Step title="Configure WebHelp transformation">
    DIM provides a pre-configured transformation scenario named **Generate Style Guide**.
    
    To use it:
    1. Click the **Configure Transformation Scenarios** button (wrench icon)
    2. Select **Generate Style Guide**
    3. Click **Edit** to review settings
    4. Ensure the output directory is set to `info-model/styleguide`
  </Step>

  <Step title="Run the transformation">
    Click **Apply associated** to execute the WebHelp transformation. Oxygen will:
    
    - Convert DITA topics to HTML
    - Generate navigation and search index
    - Apply styling and create the WebHelp interface
    - Output to `info-model/styleguide/webhelp/`
  </Step>

  <Step title="Verify the output">
    Open `info-model/styleguide/webhelp/index.html` in a browser to confirm the style guide rendered correctly.
  </Step>
</Steps>

<Warning>
The WebHelp output location must match the path configured in `info-model/rules/catalog.xml`. By default, this is `styleguide/webhelp/`. If you change the output location, update the catalog accordingly.
</Warning>

## Using Oxygen Transformation Scenarios

For convenience, you can set up Oxygen transformation scenarios for the XSLT scripts.

### Create a Gen-Rules Scenario

<Steps>
  <Step title="Open transformation scenarios">
    In Oxygen, go to **Window > Show View > Transformation Scenarios**.
  </Step>

  <Step title="Create new scenario">
    Click **New > XML transformation with XSLT**.
  </Step>

  <Step title="Configure the scenario">
    - **Name:** Generate Schematron Rules
    - **XML URL:** `${currentFileURL}` (assuming you run this on the map)
    - **XSL URL:** `${pdu}/gen-rules/gen-rules.xsl`
    - **Transformer:** Saxon-PE or Saxon-EE (XSLT 2.0 required)
    - **Output:** `${pdu}/info-model/rules/rules.sch`
  </Step>

  <Step title="Save and run">
    Click **OK** to save. Select the scenario and click **Apply associated** to run.
  </Step>
</Steps>

### Create a Gen-Mappings Scenario

Follow the same steps, but use:

- **Name:** Generate Content Mappings
- **XSL URL:** `${pdu}/gen-rules/gen-mappings.xsl`
- **Output:** `${pdu}/info-model/rules/contentCompletionElementsMap.xml`

## Command-Line Generation Workflow

For automation or CI/CD integration, use command-line tools:

```bash
#!/bin/bash

# Set paths
MAP="info-model/styleguide.ditamap"
RULES_XSL="gen-rules/gen-rules.xsl"
MAPPINGS_XSL="gen-rules/gen-mappings.xsl"
OUTPUT_DIR="info-model/rules"

# Generate Schematron schema
echo "Generating Schematron rules..."
saxon -s:$MAP -xsl:$RULES_XSL -o:$OUTPUT_DIR/rules.sch

# Generate content completion mappings
echo "Generating content mappings..."
saxon -s:$MAP -xsl:$MAPPINGS_XSL -o:$OUTPUT_DIR/contentCompletionElementsMap.xml

# Publish WebHelp (using DITA-OT)
echo "Publishing WebHelp..."
dita -i $MAP -f webhelp-responsive -o info-model/styleguide/webhelp

echo "Deliverables generated successfully!"
```

## Understanding the Generated Files

### rules.sch Structure

The generated Schematron file:

- **Includes abstract patterns** from `library.sch`
- **Instantiates patterns** with parameters from your rule definitions
- **Links to source topics** via the `see` attribute
- **Is regenerated** each time you run the transformation (do not edit manually)

### contentCompletionElementsMap.xml Structure

The generated configuration file:

- **Maps elements to topics** based on `<data>` annotations
- **Uses relative paths** within the WebHelp output
- **Includes a base URL** (`xml:base`) pointing to your WebHelp location
- **Supports filtering** with the `htmlContentFilterStylesheet` attribute
- **Is regenerated** each time you run the transformation (do not edit manually)

## Regeneration Workflow

Whenever you update your style guide:

<Steps>
  <Step title="Edit DITA topics">
    Add or modify guidelines, business rules, or metadata annotations.
  </Step>

  <Step title="Regenerate Schematron">
    Run `gen-rules.xsl` to update `rules.sch` with new or changed rules.
  </Step>

  <Step title="Regenerate mappings">
    Run `gen-mappings.xsl` to update `contentCompletionElementsMap.xml` with new annotations.
  </Step>

  <Step title="Republish WebHelp">
    Run the WebHelp transformation to update the HTML output with latest content.
  </Step>

  <Step title="Deploy to authors">
    Copy the updated deliverables to your shared framework location or publish the WebHelp to your web server.
  </Step>
</Steps>

## Deployment Options

### Local Deployment

For individual authors or small teams:

1. Place `rules.sch` and `contentCompletionElementsMap.xml` in `info-model/rules/`
2. Store WebHelp output locally
3. Configure Oxygen framework to reference these local files

### Shared Network Deployment

For teams with shared network access:

1. Deploy deliverables to a shared network drive
2. Update framework configuration to point to network paths
3. Authors reference the shared framework

### Web Deployment

For distributed teams:

1. Publish WebHelp to an internal or public web server
2. Update `catalog.xml` with the WebHelp base URL:
   ```xml
   <rewriteURI uriStartString="http://example.com/styleguide/webhelp/" 
               rewritePrefix="http://styleguide.yourcompany.com/webhelp/"/>
   ```
3. Distribute `rules.sch` and `contentCompletionElementsMap.xml` via version control or shared storage

## Troubleshooting

### XSLT Transformation Fails

**Check:**
- Using XSLT 2.0 processor (Saxon 9.x or later)
- Source map path is correct
- Stylesheet paths are correct
- Output directory exists and is writable

### Generated Schema Doesn't Validate

**Check:**
- `library.sch` is in the expected location (`info-model/rules/`)
- Include paths in generated schema are correct
- Rule parameters are valid (no typos in element names, etc.)

### Content Mappings Not Working

**Check:**
- Annotations use `audience="styleguide"`
- Topics are referenced in the style guide map
- WebHelp output exists at configured location
- `xml:base` in mappings file matches WebHelp URL

### WebHelp Links Broken

**Check:**
- File names match between DITA source (`.dita`) and HTML output (`.html`)
- Catalog configuration points to correct base path
- WebHelp was published to the expected directory

## Automated Regeneration

For continuous integration:

```yaml
# Example GitHub Actions workflow
name: Generate DIM Deliverables

on:
  push:
    paths:
      - 'info-model/**'

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Install Saxon
        run: |
          wget https://sourceforge.net/projects/saxon/files/Saxon-HE/10/Java/SaxonHE10-5J.zip
          unzip SaxonHE10-5J.zip
      
      - name: Generate Schematron
        run: |
          java -jar saxon-he-10.5.jar \
            -s:info-model/styleguide.ditamap \
            -xsl:gen-rules/gen-rules.xsl \
            -o:info-model/rules/rules.sch
      
      - name: Generate Mappings
        run: |
          java -jar saxon-he-10.5.jar \
            -s:info-model/styleguide.ditamap \
            -xsl:gen-rules/gen-mappings.xsl \
            -o:info-model/rules/contentCompletionElementsMap.xml
      
      - name: Commit generated files
        run: |
          git config user.name "GitHub Actions"
          git config user.email "actions@github.com"
          git add info-model/rules/*.sch info-model/rules/*.xml
          git commit -m "Regenerate DIM deliverables" || echo "No changes"
          git push
```
